# **Public API**

*(Ngành Bán buôn, Bán lẻ)*

![](https://cdn-kvweb.kiotviet.vn/kiotviet-website/wp-content/uploads/2025/12/07145104/KET-NOI-API-PUBLIC-API-1.jpg)

## **1. Giới thiệu**

KiotViet Public API được phát triển để hỗ trợ việc tích hợp và trao đổi dữ liệu giữa KiotViet và các nền tảng **website, thương mại điện tử, CRM…**
KiotViet Public API cung cấp cơ chế đọc và ghi các đối tượng sau:

**–** **Nhóm hàng**: lấy danh sách nhóm hàng hóa với các thông tin về tên nhóm hàng và quan hệ giữa các nhóm hàng [(2.3](#_bookmark4))

**– Hàng hóa**: lấy thông tin sản phẩm, tạo mới, sửa, xóa sản phẩm, thuộc tính của sản phẩm [(2.4](#_bookmark10))

**–** **Đặt hàng**: lấy thông tin đơn hàng, tạo đơn hàng, cập nhật và hủy đơn hàng [(](#_bookmark17)2.5)

**–** **Hóa đơn**: lấy thông tin hóa đơn, tạo hóa đơn, cập nhật và hủy hóa đơn [(2.12](#_bookmark49))

**–** **Khách hàng**: lấy danh sách khách hàng và thao tác trên thông tin khách hàng ([2.6](#_bookmark25))

**– Phiếu nhập hàng**: thông tin phiếu nhập (2.15)

**– Các API phụ trợ:**

- Danh sách chi nhánh
- Danh sách người dùng ([2.8](#_bookmark34))
- Danh sách tài khoản ngân hàng [(2.9](#_bookmark35))
- Danh sách thu khác [(2.10](#_bookmark36))
- Webhook [(2.11](#_bookmark41))
- Nhóm khách hàng ([2.13](#_bookmark55))
- Sổ quỹ [(2.14](#_bookmark57))

**Lưu ý :** Các Params có ? ở trong giá trị là những trường có thể không truyền

![](https://cdn-kvweb.kiotviet.vn/kiotviet-website/wp-content/uploads/2021/07/Api.jpg)

## **2. Chức năng**

Mục này mô tả thông tin chi tiết của từng API. Các thông tin bao gồm:

– Tên API

– Mục đích sử dụng của API

– Cấu trúc của API

– Chi tiết tham số trong request

– Nội dung response trả về

**Lưu ý:**

**–***Ngoại trừ API lấy thông tin Authentication Code và Access Token, toàn bộ các API khác đều có header trong request với thông tin:*

- **“Retailer”***:* tên gianhàng
- ​​**“Authorization”***:* Bearer {Mã Access Token}

**Ví dụ:***​*

**Retailer: taphoaxyz**

*Authorization: Bearer eyJhbGciOiJSU0EtT0FFUCIsImVu……………………..**Z31gSjq6REOpMUj3hBYBojekzw*

*– Với các hàm GET sẽ giới hạn 5000 request/1h.*
